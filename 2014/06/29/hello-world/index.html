There are so many options to consider when sitting down to build a website. These are some of the questions I had to answer when I committed mentally to building this site.

- What programming languages will you use?
- How and where will it be hosted?
- What will it cost to run?
- Will it run server side code, or require database access?

I’m not going to lie, when I first set off to build this website I had planned to use DurandalJS on the front-end, Node.json the back, maybe MongoDB for database storage, and then I went shopping for web host that could provide all these services. Since this is a personal project I didn’t want to spend too much money to host the site, so I started looking at some traditional shared virtual host providers. Most offerings where right on the money ranging from $4 - $10 dollars a month. Unfortunately most in this price range offered the standard set of host options which include:

- Control Panel Tools for site management
- PHP
- Ruby
- MySQL
- WordPress Blogging Tools

Don’t get me wrong these are all great options and for that price range a steal, it just wasn’t what I personally wantedto use. At my day job we take advantage of Amazon’s AWS services so I figured I’d take a look at whether or not it wouldit would be practical from a cost perspective. I also looked into RackSpace and Microsoft Azure cloud services as well.There is a huge competitive market in that area right now so prices all really comparable.

Amazon offers a free tier EC2 micro instance for the first year of service. This is a huge perk for this type of project and one that really enticed me. Here’s a list of the benefits you get with the free tier:

- 750 hours of EC2 running Linux/Unix Micro instance usage
- 750 hours of EC2 running Microsoft Windows Server Micro instance usage
- 750 hours of Elastic Load Balancing plus 15 GB data processing
- 30 GB of Amazon EBS Standard volume storage plus 2 million IOs and 1 GB snapshot storage
- 15 GB of bandwidth out aggregated across all AWS services
- 1 GB of Regional Data Transfer

This sounded perfect and would allow me to configure the server has I saw fit for the needs of this project. But how much would this cost after the year was up? I mean we do expect and hope that our site will be around for more than a year ;-) After reviewing the costs using the Cost Estimator provided by amazon ( http://calculator.s3.amazonaws.com/index.html ) it became clear that this might not be the best fit after all. Don’t get me wrong it wasn’t that it was overly costly especially considering the flexibility you get at that cost.

After digging around on the web for cost effective ways to use AWS services I came across some articles on building Static Sites using Amazon’s Simple Storage Service (S3) at a fraction of the cost. While doing some research on the topic of static sites I found that Amazon’s S3 service was not the only service for cheaply hosting static sites. There are even many ways to do this for free including hacks with web tools like DropBox or Google Drive, and even GitHub. Wait! what??? GitHub? Yep, that’s exactly what I ended up doing.

“But wait Michael, didn’t you say early on that one of your requirements was the ability to use Node.js and MongoDB on the server side?” You’re right, that was one of my requirements when I started this process, but after doing some homework on building static sites and the cost effectiveness they provide, it was an easy decision to make.
