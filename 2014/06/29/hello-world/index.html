Today there are so many options available to someone that wants to publish a website on the web, in the end you can boil it down to 3 considerations:

- Required Toolsets.
- Budget.
- Skill set of person building and/or maintaining.

When I first sat down to start planning how I was going to build MichaelArnel.com, I wanted to be able to showcase all of my skills and then possibly have a sandbox to play around with emerging technologies and frameworks that would be accessible to friends and peers. This is what I would consider the cadillac of options and could easily be achieved utilizing cloud services like Amazon Web Services (AWS) or Microsoft Azure.

Cloud based computing services provide an array of tools and services to fit any of your needs. With this type of service, the costs associated are typically based on a “pay for what you use” type model. This was really appealing to me early on in my process in figuring out how I wanted to host this site. Considering the services and capabilities I wanted to have access to, going the route of a traditional dedicated or shared hosting provider would have been very costly up front.

Amazon offers a free tier EC2 micro instance for the first year of service. This is a really nice offering for this type of project and really any type of project you are trying to get off the ground. Here’s a list of the benefits you get with the free tier:

- 750 hours of EC2 running Linux/Unix Micro instance usage
- 750 hours of EC2 running Microsoft Windows Server Micro instance usage
- 750 hours of Elastic Load Balancing plus 15 GB data processing
- 30 GB of Amazon EBS Standard volume storage plus 2 million IOs and 1 GB snapshot storage
- 15 GB of bandwidth out aggregated across all AWS services
- 1 GB of Regional Data Transfer

There are somethings you should keep in mind if taking an approach like Amazon’s free tier service. I listed above some benefits to the free tier, but these should also be looked at as thresholds as well. Once you start requiring more storage, bandwith, and/or additional processing power you will have moved out of the free tier level of service. Chances are however if this is the case it means your service/product is being used and the growth is welcomed. If you wanted to play around with Amazon’s cost calculator ( http://calculator.s3.amazonaws.com/index.html ) , you could estimate the expenses you would incur by using this type of service.

Of course there are other providers as well, you’re not just limited to using Amazon for cloud based services. Microsoft, RackSpace, Google and others all provide comparable offerings and the space is ultra competitive at the moment so prices are reaching new lows everyday! Feel free to explore all the other providers in your search when evaluating cloud based services for your website needs.

I expanded my research out to shared and dedicated hosting next. There are some definite advantages to going this route, and some definite dis-advantages in my opinion.

Pros:

- Cost, there are some really cheap shared hosting options out there.
- Server Maintenance and backups are handled by their professional staff.
- Uptime reliability.
- Mature control panels that make managing your site a breeze.

Cons:

- Costs, this is not a mistake, i've listed costs as a pro and a con.  There are some dedicated options that are rather costly when compared to other options.
- Flexibility, your options are usually limited to the features your web host provides. ie. Server Side tools for applications.
- Your website performance could be impacted by another customers resource utilization within the same host.

I found most hosting providers offered a similar set of services and tools for hosting your website. One of the nice benefits that almost hosts provide are plugins for WordPress, Joomla or Drupal. Simply go through the Control Panel and press a few buttons and your site will be converted to chosen platform. Using one of these platforms gives non-technical skilled individuals the ability to rapidly build their website and manage the content with little or no development skills necessary. Another nice advantage to using one of these platforms are there are literally thousands of free or small fee templates and themes that provide professional designs without needing website design skills.

As I surfed around the internet exploring all my options I kept coming across this concept of a “Static Website”, and great there for specific applications. Static websites are not alternative hosting options but more so a architectural concept for a website, which can be hosted in any of the methods already discussed.

What is a Static Website? Good question, it’s exactly as it sounds. Static websites are complete websites consisting of purley static content, meaning there’s no dynamic pages being generated or content being read from a database or some other storage mechanism. The content pages that are deployed to the server are the very same pages served to the browser. Very interesting! There are many questions and thoughts I had when I first started exploring this option. The first obvious question was why???

Static websites are light-weight and fast! Because the websites are static in content and require no intense server side processing or other costly server side applications like Databases, static websites can be hosted very cheaply. Static websites obviously won’t fit every need since they’re somewhat limited in nature to the static content they provide, but they are perfect solutions for informational sites, portfolios, blogs, etc.

This is ultimately the direction I ended up going for this website. I had to sacrifice the server-side programming tools I wanted to use, but in the end I really just needed something snappy and cost effective, static sites where a no brainer for me!
